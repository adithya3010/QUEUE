## 🌓 Dark Mode Philosophy

### Why Blue-Black Instead of Pure Black?

1. **Reduced Eye Strain**: Pure black (#000000) creates harsh contrast
2. **Better Depth**: Subtle blue tint (#0F172A) creates visual hierarchy
3. **Medical Context**: Softer, more calming for healthcare settings
4. **OLED Friendly**: Still dark enough for battery savings

### Color Temperature

- **Light Mode**: Slightly cool (bluish white) - Professional, clean
- **Dark Mode**: Deep blue-black - Calming, reduces harsh contrast

---

## 💡 Pro Tips

1. **Always test both modes** - Preview every component in light AND dark
2. **Use semantic colors** - `success-*` not arbitrary greens
3. **Maintain hierarchy** - Use neutral-900 → neutral-700 → neutral-500 for text importance
4. **Add transitions** - `transition-colors duration-200` for smooth theme switches
5. **Check contrast** - Use browser DevTools to verify readability

## 🔄 Theme Toggle Behavior

```
User clicks theme toggle:
┌─────┬─────┬─────┐
│ ☀️  │ 🖥️  │ 🌙  │
└─────┴─────┴─────┘
  │     │     │
Light System Dark
Mode  Theme Mode
```

- **Light**: Forces light mode
- **System**: Follows OS preference
- **Dark**: Forces dark mode

Preference is saved to localStorage and persists across sessions.
